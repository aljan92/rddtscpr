RapidAPI-Absicherung & Sandbox

Verbinde und schütze deine API-Endpunkte für den RapidAPI Marketplace

Im Sandbox-Modus wird die Prüfung des `Proxy-Secret` Headers übersprungen. Du kannst die API direkt aufrufen und über den `X-RapidAPI-Subscription` Header beliebige Abonnements (Basic, Pro, Ultra) simulieren.

Sandbox-Modus ist AKTIV. Es ist kein Proxy-Secret-Schutz auf den öffentlichen Endpunkten aktiv.

Das geheime Token aus deinem RapidAPI Entwickler-Dashboard. Schützt deine API vor Zugriffen außerhalb von RapidAPI, sobald der Sandbox-Modus deaktiviert ist.

📖 Dokumentation & Funktionsweise

Was macht das Proxy-Secret?

Sobald ein Nutzer deine API über den RapidAPI-Marktplatz aufruft, leitet RapidAPI den Request an deinen VPS weiter und hängt automatisch den Header `X-RapidAPI-Proxy-Secret` mit dem in RapidAPI hinterlegten Schlüssel an. Wenn der Sandbox-Modus deaktiviert ist, weist dein Server jeden Request ohne diesen korrekten Header mit einem 403 Forbidden ab.

Abonnements simulieren (Sandbox)

Wenn der Sandbox-Modus aktiv ist, kannst du mit Tools wie `curl` oder Postman die Reaktion der API auf verschiedene Abos testen, indem du den Header `X-RapidAPI-Subscription` mitsendest:

  • Basic: `/v1/post-comments` schlägt fehl, wenn `load_more=true` aufgerufen wird (für Free-Pläne gesperrt).
  • Pro / Ultra: Alle Features (inkl. `load_more`) funktionieren unbeschränkt.
